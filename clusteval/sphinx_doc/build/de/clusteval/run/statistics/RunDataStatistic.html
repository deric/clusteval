RunDataStatistic¶

public abstract class RunDataStatistic extends Statistic¶

A run-data statistic is a Statistic, which summarizes relationships of clustering run results and data set properties. Run-data statistics are assessed by a RunDataAnalysisRun.

A run-data statistic MyRunDataStatistic can be added to ClustEval by

  1. extending the class RunDataStatistic with your own class MyRunDataStatistic. You have to provide your own implementations for the following methods, otherwise the framework will not be able to load your class.
  • RunDataStatistic(Repository, boolean, long, File) : The constructor for your run-data statistic. This constructor has to be implemented and public.
  • :java:ref:`RunDataStatistic(MyRunDataStatistic)`q : The copy constructor for your run-data statistic. This constructor has to be implemented and public.
  • Statistic.getAlias() : See Statistic.getAlias().
  • Statistic.parseFromString(String) : See Statistic.parseFromString(String).
  1. extending the class RunDataStatisticCalculator with your own class MyRunDataStatisticCalculator . You have to provide your own implementations for the following methods.
  • RunDataStatisticCalculator(Repository, long, File, DataConfig) : The constructor for your run-data statistic calculator. This constructor has to be implemented and public.
  • RunDataStatisticCalculator(MyRunDataStatisticCalculator) : The copy constructor for your run-data statistic calculator. This constructor has to be implemented and public.
  • RunDataStatisticCalculator.calculateResult(): See StatisticCalculator.calculateResult().
  • StatisticCalculator.writeOutputTo(File): See StatisticCalculator.writeOutputTo(File).
  1. Creating a jar file named MyRunDataStatisticCalculator.jar containing the MyRunDataStatistic.class and MyRunDataStatisticCalculator.class compiled on your machine in the correct folder structure corresponding to the packages:
  • de/clusteval/run/statistics/MyRunDataStatistic.class
  • de/clusteval/run/statistics/MyRunDataStatisticCalculator.class
  1. Putting the MyRunDataStatistic.jar into the run-data statistics folder of the repository:
  • <REPOSITORY ROOT>/supp/statistics/rundata
  • The backend server will recognize and try to load the new run statistics automatically the next time, the RunDataStatisticFinderThread checks the filesystem.
Author:Christian Wiwie
