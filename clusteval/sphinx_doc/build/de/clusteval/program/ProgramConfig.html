ProgramConfig¶

public class ProgramConfig extends RepositoryObject¶

A program configuration encapsulates a program together with options and settings.

A program configuration corresponds to and is parsed from a file on the filesystem in the corresponding folder of the repository (see Repository.programConfigBasePath and ProgramConfigFinder).

There are several options, that can be specified in the program configuration file (see parseFromFile(File)).

Author:Christian Wiwie

compatibleDataSetFormats¶

protected List<DataSetFormat> compatibleDataSetFormats¶

This list holds all dataset formats that are compatible with the encapsulated program, i.e. input formats this program is able to read.

expectsNormalizedDataSet¶

protected boolean expectsNormalizedDataSet¶

This boolean indicates, whether the encapsulated program requires a normalized dataset, i.e. similarities between 0 and 1. This is then handled before the data is passed to the clustering method.

invocationFormat¶

protected String invocationFormat¶

This is the default invocation line used to invoke the program, when this program configuration is used together with some data configuration.

This invocation line is used, if

  • there is a goldstandard in the data configuration
  • the run is not of type parameter optimization

The maximal time this program config should be executed. The execution is terminated when this time is reached.

Instantiates a new program config.

Parameters:
  • repository – The repository this program configuration should be registered at.
  • register – A boolean indicating whether to register this program configuration at the repository.
  • changeDate – The change date of this program configuration is used for equality checks.
  • absPath – The absolute path of this program configuration.
  • program – The program this program configuration belongs to.
  • outputFormat – The output format of the program.
  • compatibleDataSetFormats – A list of compatible dataset formats of the encapsulated program.
  • invocationFormat – The invocation line for runs with goldstandard and without parameter optimization
  • invocationFormatWithoutGoldStandard – The invocation line for runs without goldstandard and without parameter optimization
  • invocationFormatParameterOptimization – The invocation line for runs with goldstandard and with parameter optimization
  • invocationFormatParameterOptimizationWithoutGoldStandard – The invocation line for runs without goldstandard and with parameter optimization
  • params – The parameters of the program.
  • optimizableParameters – The parameters of the program, that can be optimized.
  • expectsNormalizedDataSet – Whether the encapsulated program requires normalized input.
