RepositoryConfig¶

public class RepositoryConfig¶

A repository configuration determines certain settings and options for a Repository and also for the complete backend. This includes for example whether an sql database should be used or how often the supervising threads of the repository should scan for changes.

Author:Christian Wiwie

parseFromFile¶

public static RepositoryConfig parseFromFile(File absConfigPath)¶

This method parses a repository configuration from the file at the given absolute path.

A repository configuration contains several sections and possible options:

  • [mysql]
  • [threading]
  • NameOfTheThreadSleepTime: Sleeping time of the thread ‘NameOfTheThread’. This option can be used to control the frequency, with which the threads check for changes on the filesystem.
Parameters:
  • absConfigPath – The absolute path of the repository configuration file.
Throws:
  • RepositoryConfigurationException –
  • RepositoryConfigNotFoundException –
Returns:

The parsed repository configuration.
