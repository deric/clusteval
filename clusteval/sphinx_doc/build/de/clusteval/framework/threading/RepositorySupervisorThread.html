RepositorySupervisorThread¶

public class RepositorySupervisorThread extends SupervisorThread¶

A type of supervisor thread which creates the standard set of threads responsible for the initialization of a standard Repository. In detail threads of the following type are created, started and kept alive:

  • DataSetFormatFinderThread: A thread which checks Repository.dataSetFormatBasePath for new dataset formats (see DataSetFormat).
  • DataSetTypeFinderThread: A thread which checks Repository.dataSetTypeBasePath for new dataset types (see DataSetType).
  • DataSetFinderThread: A thread which checks Repository.basePath for new datasets (see DataSet).
  • DistanceMeasureFinderThread: A thread which checks Repository.distanceMeasureBasePath for new distance measures (see DistanceMeasure).
  • DataStatisticFinderThread: A thread which checks Repository.dataStatisticBasePath for new data statistics (see DataStatistic).
  • RunStatisticFinderThread: A thread which checks Repository.runStatisticBasePath for new run statistics (see RunStatistic).
  • RunDataStatisticFinderThread: A thread which checks Repository.runDataStatisticBasePath for new run data statistics (see RunDataStatistic).
  • RunResultFormatFinderThread: A thread which checks Repository.runResultFormatBasePath for new runresult formats (see RunResultFormat).
  • ClusteringQualityMeasureFinderThread: A thread which checks Repository.clusteringQualityMeasureBasePath for new clustering quality measures (see ClusteringQualityMeasure).
  • ParameterOptimizationMethodFinderThread: A thread which checks Repository.parameterOptimizationMethodBasePath for new parameter optimization methods (see DataSetFormat).
  • DataSetConfigFinderThread: A thread which checks Repository.dataSetConfigBasePath for new dataset configurations (see DataSetConfig).
  • GoldStandardConfigFinderThread: A thread which checks Repository.goldStandardConfigBasePath for new goldstandard configurations (see GoldStandardConfig).
  • DataConfigFinderThread: A thread which checks Repository.dataConfigBasePath for new data configurations (see DataConfig).
  • RProgramFinderThread: A thread which checks Repository.programBasePath for new RPrograms (see RProgram).
  • ProgramConfigFinderThread: A thread which checks Repository.programConfigBasePath for new program configurations (see ProgramConfig).
  • DataSetGeneratorFinderThread: A thread which checks Repository.dataSetGeneratorBasePath for new dataset generators (see DataSetGenerator).
  • RunFinderThread: A thread which checks Repository.runBasePath for new runs (see Run).
  • RunSchedulerThread: A thread which is responsible for scheduling, starting and terminating runs. It can be controlled with its RunSchedulerThread.schedule(String,String), RunSchedulerThread.scheduleResume(String,String) and RunSchedulerThread.terminate(String,String) methods.

If the boolean parameter checkForRunResults is true, additionally the following threads are started and kept alive:

  • RunResultFinderThread: A thread which checks Repository.runResultBasePath for new run results (see RunResult).
Author:Christian Wiwie

  • repository – The repository this thread belongs to.
  • threadSleepTimes – The sleep times of the created threads.
